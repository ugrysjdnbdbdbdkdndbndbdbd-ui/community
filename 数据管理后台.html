📖 使用说明

  • 推荐打开方式：在项目目录执行 npx serve .，浏览器访问 http://localhost:3000/数据管理后台.html，以加载 SQL.js 启用完整功能（编辑、同步、备份等）。
  • 仅用 file 打开：将使用本地存储模式，可查看、导出数据；编辑与同步需通过上述 HTTP 方式打开。
  • 数据来源：community_cases.js 为案例主数据；本后台可同步、编辑并导出，与「社区推导看板」通过 localStorage 联动。

📍 推荐操作顺序

按左侧导航从上到下操作：数据同步（或数据导入）→ 案例管理 / 数据采集 → 数据质量 / 信息增密（可再按建议采集）→ 数据导出。

建议先完成「数据同步」再采集；信息增密报告可指导「按建议采集」补全缺失项。

使用说明

  • 实效性：依据案例 timeline 最后年份与当前年份差，标记 fresh / normal / stale，便于优先更新过期案例。
  • 信息增密：对每条案例检查是否具备 user_metrics、business_model、key_metrics 及关键子字段，生成密度分与缺失项清单。
  • 核心案例增密：针对预定义核心案例列表检查完整度，导出待增密优先级列表。
  • 多轮增密：按第1轮（基础数据）、第2轮（历程与竞争）、第3轮（产品与技术）、第4轮（理论深度）输出每案例达标情况与缺失项，便于分轮次排期增密。详见文档《案例库信息增密专项-多轮增密方案》。
  • 根据建议采集：运行报告后，可点击「对建议案例执行采集」对缺失 user_metrics / business_model 的案例执行用户数据与商业数据采集（会跳转到数据采集页并写采集日志）。
  • 导出 CSV/JSON 后可配合线下编辑或脚本批量补数，再通过「数据同步」或直接编辑 community_cases.js 回写。
