# 事故复盘：中文引号导致JavaScript语法错误

## 事故时间
2024年（具体日期）

## 事故描述
**症状：**
- 浏览器控制台出现大量 `SyntaxError: Invalid or unexpected token` 错误
- `wisdomCapsules` 对象无法加载，显示 `wisdomCapsules is undefined`
- 所有理论卡片无法显示
- 错误集中在 `wisdom_capsules.js` 文件

**影响范围：**
- 所有理论相关页面（logic, space, gov, econ等）的理论卡片全部无法显示
- 理论关联图谱功能无法正常工作

## 根本原因

### 1. 直接原因
`wisdom_capsules.js` 文件中的 `content` 字段使用了**中文引号**（"、"）而不是英文引号（"）。

### 2. 技术原因
在JavaScript字符串中：
- **英文引号** `"` 是字符串的边界字符，需要转义（`\"`）才能在字符串内容中使用
- **中文引号** `"` 和 `"` 是普通Unicode字符，但在JavaScript解析器中，如果出现在字符串中间，可能导致解析错误
- 当字符串中包含中文引号时，JavaScript解析器可能认为字符串提前结束，导致后续字符被当作代码解析，从而产生语法错误

### 3. 为什么会出现这个问题
在添加或编辑 `wisdom_capsules.js` 文件时，直接从中文输入法复制粘贴了包含中文引号的文本，没有意识到这些引号会导致JavaScript语法错误。

## 错误示例

### 错误代码
```javascript
content: "不要试图逼迫 90% 的人生产内容，而是要服务好那 1% 的核心创作者，同时降低 9% 用户的互动门槛（如"一键三连"）。",
```

### 正确代码
```javascript
content: "不要试图逼迫 90% 的人生产内容，而是要服务好那 1% 的核心创作者，同时降低 9% 用户的互动门槛（如\"一键三连\"）。",
```

## 修复过程

### 步骤1：识别问题
- 通过浏览器控制台错误信息定位到 `wisdom_capsules.js` 文件
- 使用 `node --check` 命令检查语法错误
- 发现错误集中在包含中文引号的 content 字段

### 步骤2：初步修复尝试
- 尝试将中文引号替换为转义的英文引号
- **问题**：在替换过程中出现了过度转义，导致 `\"` 变成了 `\\"`（两个反斜杠）

### 步骤3：深度修复
使用字节级别的替换：
```python
# 将文件中的 \\"（两个反斜杠+引号）替换为 \"（一个反斜杠+引号）
old_bytes = bytes([0x5C, 0x5C, 0x22])  # \\"
new_bytes = bytes([0x5C, 0x22])        # \"
content = content.replace(old_bytes, new_bytes)
```

### 步骤4：验证
- 使用 `node --check` 验证语法正确性
- 确认 `wisdomCapsules` 对象可以正常加载

## 修复时间线
1. **问题发现**：用户报告理论卡片不显示，控制台大量错误
2. **问题定位**：约30分钟（查找错误原因，定位到中文引号问题）
3. **初步修复**：约1小时（尝试各种替换方法，遇到转义问题）
4. **最终修复**：约30分钟（使用字节级别替换，彻底解决问题）
5. **验证确认**：约10分钟（验证语法正确性）

**总计耗时**：约2小时

## 经验教训

### 1. 预防措施

#### 代码规范
- ✅ **严禁在JavaScript代码中使用中文引号**
- ✅ 所有字符串内容中的引号必须使用转义的英文引号（`\"` 或 `\'`）
- ✅ 在编辑包含中文字符的JavaScript文件时，注意检查引号类型

#### 开发流程
- ✅ **代码提交前必须进行语法检查**
  ```bash
  node --check your-file.js
  ```
- ✅ 使用编辑器配置自动检测中文标点符号
- ✅ 在代码审查中特别注意字符串内容的引号类型

#### 测试流程
- ✅ 每次修改 `wisdom_capsules.js` 后，必须在浏览器中验证功能正常
- ✅ 检查浏览器控制台是否有语法错误
- ✅ 验证关键对象（如 `wisdomCapsules`）是否正确加载

### 2. 工具和脚本

#### 语法检查脚本
创建自动检查脚本，在提交前运行：
```bash
#!/bin/bash
# check-js-syntax.sh

echo "检查JavaScript语法..."

if node --check wisdom_capsules.js; then
    echo "✅ wisdom_capsules.js 语法正确"
else
    echo "❌ wisdom_capsules.js 语法错误，请修复后再提交"
    exit 1
fi

if node --check community_cases.js; then
    echo "✅ community_cases.js 语法正确"
else
    echo "❌ community_cases.js 语法错误，请修复后再提交"
    exit 1
fi

echo "✅ 所有JavaScript文件语法检查通过"
```

#### 中文引号检测脚本
已创建 `check-chinese-quotes.py` 脚本，用于检测JavaScript文件中的中文引号。

**注意：** 该脚本会检测所有包含中文引号字符（"、"、'、'）的行。如果文件中的引号已经正确转义为 `\"`，脚本可能会显示这些行，但这是正常的——重要的是确保这些引号在JavaScript字符串中是正确的转义形式。

**使用方法：**
```bash
python3 check-chinese-quotes.py [filename]
```

如果检测到中文引号，请检查：
1. 这些引号是否在字符串内容中
2. 如果是，应该使用转义的英文引号 `\"` 替代
3. 运行 `node --check` 验证语法是否正确
