# 数据管理后台使用说明书

**产品名称：** 案例库数据管理后台 - 社区产品推导系统  
**入口文件：** `数据管理后台.html`  
**版本：** 2026

---

## 一、产品简介

数据管理后台是**社区洞察全景 Akashic** 的案例库管理与维护中心，用于案例数据的**同步、导入、编辑、采集、质量检查、信息增密**以及**导出与统计分析**。数据来源以 `community_cases.js` 为主；后台支持 SQL.js 本地数据库与 localStorage 双模式，可与「社区推导看板」通过 localStorage 联动展示。

**适用角色：** 运营、数据管理员、产品经理（负责案例库维护与增密）。

---

## 二、访问方式与运行模式

### 2.1 推荐打开方式（完整功能）

在项目根目录执行本地 HTTP 服务后，用浏览器访问后台页面，以加载 SQL.js（WASM），启用**编辑、同步、备份**等完整功能：

```bash
# 在项目目录下执行
npx serve .
```

浏览器访问：**`http://localhost:3000/数据管理后台.html`**

- 支持：数据同步、案例增删改、批量操作、数据采集（含后端 API）、数据导出、操作历史等全部功能。

### 2.2 仅用 file 协议打开

若直接双击打开 `数据管理后台.html`（`file://` 协议）：

- **可用**：数据概览、案例列表查看、数据导出、图标管理查看等。
- **受限**：数据库未初始化时，**案例编辑、删除、数据同步**等写操作不可用；采集会走本地回退逻辑。
- 页面会提示「仅读模式」，并说明需通过 HTTP 方式打开以启用完整功能。

### 2.3 从主系统进入

在「社区推导看板」中：**工具箱** → **数据管理后台**，会在新标签页打开 `数据管理后台.html`（需保证路径正确，通常与主看板同域或同目录）。

## 四、推荐操作顺序

按左侧导航**从上到下**的推荐流程：

1. **数据同步**（或数据导入）→ 先让本地有案例数据（硬编码同步或文件导入）。
2. **案例管理 / 数据采集** → 维护案例列表、补全字段，或执行自动采集。
3. **数据质量 / 信息增密** → 运行检查与增密报告，按建议补采或编辑。
4. **数据导出** → 导出为 Excel/CSV/JSON，供汇报或备份。

信息增密报告可指导「按建议采集」补全缺失的 user_metrics / business_model。

## 五、各功能模块说明

### 5.1 📊 数据概览

- **统计卡片**：总案例数、已采集数据数、待更新数、数据完整度百分比。
- **图表**：数据采集趋势、数据完整度分布。
- **最近更新的案例**：列表展示最近更新的案例条目。
- **使用说明与推荐顺序**：首屏文案说明打开方式（HTTP vs file）、数据来源及推荐操作顺序。

### 5.2 🔄 数据同步

- **同步硬编码数据**：将 `community_cases.js` 中的案例数据同步到本地数据库，并显示进度条与状态文案。**首次使用或数据库未初始化时建议先执行此项。**
- **同步到云端**：将当前本地数据推送到云端（需后端/配置支持）。
- **从云端拉取**：从云端拉取最新数据到本地。
- **同步日志**：查看本次同步过程的日志输出。

### 5.3 📥 数据导入

支持三种格式，点击对应区域或触发隐藏的 file 选择框：

| 格式   | 说明           |
|--------|----------------|
| Excel  | .xlsx / .xls   |
| CSV    | .csv           |
| JSON   | .json           |

- 导入结果在页面下方 **导入结果** 区域显示。
- 若存在同 `case_key`，具体覆盖/跳过策略以当前版本界面或二次确认弹窗为准；关键操作会有确认提示。

### 5.4 📦 案例管理

- **列表**：表格展示案例图标、案例标识（case_key）、名称、分类/标签、状态、5D、三块、数据完整度、质量（预留）、网站、最后更新、操作。
- **筛选**：搜索框 + 分类下拉（强关系/算法/私密/公开/垂直/知识/游戏/其他）+ 状态下拉（活跃/未活跃/已归档），实时过滤列表。
- **批量操作**：勾选多行后显示批量操作栏，支持**批量编辑**、**批量删除**、**取消选择**；表头复选框可全选/取消全选。
- **单条操作**：编辑、删除等（具体以表格内按钮为准）。
- **新增案例**：点击「+ 新增案例」打开新增表单。
- **分页**：每页 20/50/100 条可调，显示当前范围与总条数，上一页/下一页切换。

若为仅读模式（如 file 打开且 DB 未初始化），会显示黄色提示条，说明需通过 HTTP 打开或先执行「数据同步」以启用编辑/删除。

### 5.5 🖼️ 图标管理

- **用途**：为每个案例配置在主看板与列表中展示的图标，配置保存在本地（localStorage），主看板优先使用此处设置。
- **配置类型**：本地 `icons/` 文件夹中的图片（相对路径 `icons/文件名`）、Emoji（单字符）、或图片 URL（http(s) 或 data: 开头）。
- **操作**：搜索案例 key 或名称；表格中可为每条案例选择/更换图标；**批量用本地图标匹配**、**导出图标配置**。
- **本地图标选择弹窗**：从项目 `icons/` 目录列出的文件中选择，支持按文件名搜索。

### 5.6 🔍 数据采集

- **5D 基因指纹数据**：从产品分析中提取 5D 评分，点击「开始采集」执行。
- **用户数据**：采集 MAU、DAU、留存率等，点击「开始采集」。
- **商业数据**：采集 ARPU、LTV、收入等，点击「开始采集」。
- **按信息增密建议采集**：对「信息增密」或「核心案例增密」报告中缺失 user_metrics / business_model 的案例，执行用户数据 + 商业数据采集（需先到「信息增密」页运行对应报告）。
- **采集日志**：右侧显示采集过程日志，支持「清空日志」。  
- **说明**：采集会先请求后端健康检查（如 `/api/health`）；若后端不可用，将使用本地数据回退并给出单条提示，避免逐条超时报错。

### 5.7 🔍 数据质量

- **开始检查**：执行数据完整性、准确性、一致性检查。
- **质量统计卡片**：总问题数、错误数、警告数、数据完整度。
- **问题列表**：按严重程度（错误/警告/信息）、状态（未处理/已修复/已忽略）筛选，列表展示问题明细。
- 列表中的「质量」列预留与质量模块对接，二期可显示质量状态或徽标。

### 5.8 📈 信息增密

- **三个报告维度**：  
  - **⏱ 实效性**：基于 timeline 最后年份判断数据是否过时（fresh / normal / stale），便于优先更新过期案例。  
  - **📊 信息增密**：检查 user_metrics、business_model、key_metrics 及关键子字段，输出密度分与缺失项。  
  - **⭐ 核心案例增密**：针对预定义核心案例检查完整度，输出待增密优先级列表。
- **操作**：每个维度可单独「运行」，或使用「一键运行全部」。
- **报告结果**：下拉切换报告类型（实效性/信息增密/核心案例增密），表格展示结果；支持**导出 CSV**、**导出 JSON**。
- **根据建议采集**：运行报告后，可点击「对建议案例执行采集」，对缺失项案例执行用户+商业数据采集（会写采集日志）。
- 导出 CSV/JSON 后可线下或脚本批量补数，再通过数据同步或编辑 `community_cases.js` 回写。

### 5.9 📤 数据导出

- **格式**：导出 **Excel**（.xlsx）、**CSV**、**JSON**。
- **导出选项**（勾选控制）：5D 基因指纹数据、用户数据、商业数据、增长数据；导出文件中会包含数据完整度等与列表一致的列（以当前版本为准）。

### 5.10 📊 数据分析

- **筛选**：按分类、状态筛选；统计维度可选「按分类」「按状态」「按完整度」。
- **统计卡片**：案例总数、平均完整度、有 5D 数据条数、有用户数据条数。
- **图表**：分类分布、状态分布、5D 评分分布等。
- **导出报告**：将当前统计与图表导出为报告文件。

### 5.11 📜 操作历史

- **筛选**：操作类型（新增/编辑/删除/导入/导出）、对象类型（案例/5D评分/用户数据/商业数据）、案例 Key、时间范围（全部/今天/最近一周/最近一月）。
- **操作记录列表**：分页展示，显示每条操作的类型、对象、时间等。
- **清理旧记录**：清理历史记录以释放空间。
- **导出记录**：将当前筛选结果导出。

### 5.12 ⚙️ 系统设置

- **数据存储方式**：本地存储（LocalStorage）/ SQLite 数据库 / JSON 文件（以实际选项为准）。
- **自动采集频率**：每天 / 每周 / 每月 / 手动。
- **数据备份**：点击「立即备份」执行备份（具体备份目标以实现为准）。

## 六、后端服务与采集 API（可选）

- 若本机运行 Node 采集服务（如 `server/server.js`），顶部**后端服务控制**可用来检查状态、启动、停止、重启服务，以及查看**服务器日志**（弹窗或侧边面板）。
- 采集请求的 **API 基地址** 默认为当前页面的 origin；若用 file 打开则回退为 `http://localhost:3000`。可在页面或脚本中通过 `window.CASES_API_BASE` 覆盖，便于不同环境部署。
- 健康检查接口：`/api/health`，用于采集前判断后端是否可用；不可用时采集走本地回退并给出提示。

## 七、数据完整度说明

- 全站统一规则（与优化二期一致）：**5D 40% + 用户数据 30% + 商业 15% + 关键指标 15%**。
- 当数据库与 `community_cases.js` 均有数据时，完整度取两者**较大值**，避免 DB 覆盖掉更高的 JS 完整度。
- 列表、详情、导出中的「数据完整度」与此口径一致。

## 八、常见问题

**Q：为什么编辑/删除案例是灰色的或提示仅读？**  
A：多为 file 协议打开且数据库未初始化。请用 `npx serve .` 后访问 `http://localhost:3000/数据管理后台.html`，并先执行「数据同步」中的「同步硬编码数据」。

**Q：采集时满屏「API 不可用」？**  
A：当前版本会在采集前做健康检查，若后端不可用则改为本地回退并只给一条汇总提示。确认本机是否启动了采集服务，以及 `CASES_API_BASE` 是否指向正确地址。

**Q：数据会丢吗？**  
A：写操作会写入本地数据库或 localStorage；清除浏览器数据会清空本地状态。重要数据建议定期通过「数据导出」备份为 Excel/CSV/JSON。

**Q：主看板看不到新案例或图标？**  
A：主看板与后台通过 localStorage 及 `community_cases.js` 联动。确保后台「数据同步」或导出后已更新 `community_cases.js`（或主看板所读数据源）；图标以「图标管理」中配置为准。
