## 一、发布方式：GitHub Pages

本项目已配置为可通过 **GitHub Pages** 发布为静态站点，访问地址与仓库一一对应。

### 1. 匹配的网址（启用后）

| 用途       | 网址 |
|------------|------|
| **站点首页**（自动跳转到看板） | `https://ugrysjdnbdbdbdkdndbndbdbd-ui.github.io/community/` |
| **社区推导看板** | `https://ugrysjdnbdbdbdkdndbndbdbd-ui.github.io/community/社区推导看板.html` |
| **数据管理后台** | `https://ugrysjdnbdbdbdkdndbndbdbd-ui.github.io/community/数据管理后台.html` |

根路径 `/` 会通过 `index.html` 自动跳转到「社区推导看板」。

### 2. 在 GitHub 上启用 Pages（首次发布）

1. 打开仓库：  
   **https://github.com/ugrysjdnbdbdbdkdndbndbdbd-ui/community**
2. 点击 **Settings（设置）** → 左侧 **Pages**。
3. 在 **Build and deployment** 中：
   - **Source** 选 **Deploy from a branch**。
   - **Branch** 选 `main`，**Folder** 选 **/ (root)**。
4. 点击 **Save**。
5. 等待 1–2 分钟，然后访问：  
   **https://ugrysjdnbdbdbdkdndbndbdbd-ui.github.io/community/**

### 3. 后续更新

- 每次向 `main` 分支 **push** 后，GitHub 会自动重新部署。
- 无需额外构建，当前为纯静态部署（HTML/JS/图标等）。

## 二、项目内与网址相关的文件

| 文件/目录        | 作用 |
|------------------|------|
| `index.html`     | 站点根路径入口，跳转到「社区推导看板」 |
| `社区推导看板.html` | 主应用页面 |
| `数据管理后台.html` | 数据管理后台页面 |
| `community_cases.js` / `wisdom_capsules.js` | 数据与文案，相对路径引用 |
| `icons/`         | 图标资源，相对路径引用 |
| `.nojekyll`      | 告知 GitHub Pages 不按 Jekyll 处理，保证所有文件被部署 |

所有资源均使用**相对路径**，部署到任意子路径（如 `/community/`）均可正常访问。

## 三、自定义域名（可选）

若需使用自己的域名（例如 `akashic.yourdomain.com`）：

1. 在仓库根目录新建文件 **CNAME**，内容写你的域名（一行，无协议、无路径），例如：  
   `akashic.yourdomain.com`
2. 在域名服务商处添加 **CNAME 记录**，将上述主机名指向：  
   `ugrysjdnbdbdbdkdndbndbdbd-ui.github.io`
3. 在 GitHub 仓库 **Settings → Pages** 中确认 **Custom domain** 显示该域名并保存。

保存后，访问该域名即会匹配到当前 GitHub Pages 站点。

## 四、注意事项

- **仅前端静态资源**会通过 GitHub Pages 发布；`server/` 下的 Node 服务需自行部署到支持 Node 的平台（如 Railway、Render 等），不在本说明范围内。
- 若需后端 API，请参考 [网站配置SOP.md](./网站配置SOP.md) 在自有服务器或云服务上部署 `server/`。
