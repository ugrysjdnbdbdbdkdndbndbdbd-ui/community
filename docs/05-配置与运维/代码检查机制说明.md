# 代码检查机制说明

## 📋 概述

本项目建立了完整的代码检查机制，确保代码质量和一致性。所有检查脚本都在项目根目录。

---

## 🔍 检查脚本

### 1. JavaScript语法检查

**脚本**：`check-js-syntax.sh`

**功能**：
- 检查所有JavaScript文件的语法正确性
- 使用Node.js的`--check`选项进行语法验证

**使用方法**：
```bash
bash check-js-syntax.sh
```

**检查的文件**：
- `wisdom_capsules.js`
- `community_cases.js`

**输出**：
- ✅ 语法正确：显示"语法正确"
- ❌ 语法错误：显示错误信息和位置

---

### 2. 中文引号检查

**脚本**：`check-chinese-quotes.py`

**功能**：
- 检查JavaScript文件中是否包含未转义的中文引号
- 只检查`content:`字段中的引号

**使用方法**：
```bash
python3 check-chinese-quotes.py [filename]
# 或检查所有文件
python3 check-chinese-quotes.py
```

**检查的引号类型**：
- `"` - 中文左双引号
- `"` - 中文右双引号
- `'` - 中文左单引号
- `'` - 中文右单引号

**输出**：
- ✅ 未发现问题：显示"未发现中文引号问题"
- ❌ 发现问题：显示问题位置和内容预览

**注意**：
- 此检查可能产生误报（如果引号已在转义序列中）
- 建议结合语法检查一起使用

---

### 3. 提交前综合检查

**脚本**：`pre-commit-check.sh`

**功能**：
- 综合所有检查脚本
- 在提交代码前运行，确保代码质量

**使用方法**：
```bash
bash pre-commit-check.sh
```

**检查项**：
1. JavaScript语法检查
2. 中文引号检查
3. 文件大小检查（>1MB警告）

**退出码**：
- `0` - 所有检查通过或只有警告
- `1` - 发现错误，阻止提交

---

## 🚀 集成到开发流程

### 方法1：手动运行

在提交代码前，手动运行检查脚本：
```bash
bash pre-commit-check.sh
```

### 方法2：Git Hooks（推荐）

将检查脚本集成到Git pre-commit hook：

```bash
# 创建git hooks目录（如果不存在）
mkdir -p .git/hooks

# 创建pre-commit hook
ln -s ../../pre-commit-check.sh .git/hooks/pre-commit

# 或直接复制
cp pre-commit-check.sh .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

这样每次`git commit`时都会自动运行检查。

## 📝 检查标准

### 必须通过的检查（错误）

- ✅ JavaScript语法必须正确
- ✅ 不能有语法错误

### 建议修复的检查（警告）

- ⚠️ 中文引号问题（可能是误报）
- ⚠️ 文件过大（>1MB）

## 🎯 最佳实践

1. **提交前必检**：每次提交代码前运行检查脚本
2. **及时修复**：发现错误立即修复，不要累积
3. **定期检查**：即使没有修改，也定期运行检查
4. **团队协作**：确保团队成员都使用相同的检查标准

---

## 📌 注意事项

1. **中文引号检查**：
   - 可能产生误报（如果引号已在转义序列中）
   - 建议结合语法检查一起判断
   - 如果语法检查通过，可以忽略中文引号警告

2. **文件大小**：
   - 大文件（>1MB）会显示警告
   - 不影响提交，但建议优化

3. **备份文件**：
   - 修复脚本会自动创建备份
   - 备份文件格式：`filename.backup_YYYYMMDD_HHMMSS`
   - 可以手动删除旧备份
